Pre-call setup (do this before the meeting)

1Create the Hunt Chrysler tenant

  • Admin panel → New Tenant → fill in dealership name + Mil's email
  • Set tier = gold
  • Set seat allowance = 10
  • Save. Confirm tenant_id created.

2Set the lead intake email

  • In admin panel for the new tenant: set lead_intake_email = huntchrysler@firstfinancialcanada.com
  • Verify Gmail Workspace forwards huntchrysler@firstfinancialcanada.com to firstfinancialcanada@gmail.com (the IMAP-monitored mailbox)
  • Send a test email to verify routing → check that an inbound shows in the lead intake log

3Pre-import Hunt's inventory

  • Log in as Mil's account (or use admin impersonation)
  • Install the First Fin Inventory Importer Chrome extension
  • Visit huntchrysler.ca/vehicles/used → click Scan → wait ~5–7 minutes
  • Then visit huntchrysler.ca/vehicles/new → click Scan → wait ~5–7 minutes
  • Sync both with ADD mode
  • Confirm in Inventory tab — should be ~200–300 vehicles with 18–25 photos each
Hunt Chrysler is on the Convertus / Achilles platform (same as Milton Chrysler). The deep photo scan will work without modification — same selectors, same Cloudflare behavior, same recipe.

4Provision a Twilio number for Sarah

  • Twilio Console → Phone Numbers → Buy a number in Mil's local area (Milton 905 area code)
  • Set webhook to point at https://app.firstfinancialcanada.com/twilio/sms-incoming
  • Update tenant settings: twilioNumber = +19058782580 (or whatever number you bought)
  • Test — send an SMS to that number, confirm it lands in CRM and triggers Sarah's reply

8Wire AutoTrader / Kijiji / CarCostCanada / TAQ to send leads to First Fin

Recommended starter approach (safer, dual-recipient):

Mil keeps his existing AutoTrader / Kijiji / CarCostCanada / TAQ setup 100% intact, just adds huntchryslerfiat@firstfinancialcanada.com as an additional lead recipient in each provider's dashboard. Both inboxes get every lead. Zero risk of "did a lead get lost?" panic — his old workflow is untouched.

For each provider, the path is roughly:

  • AutoTrader.ca dealer dashboard → Settings / Lead Distribution → Add Email Recipient
  • Kijiji Autos dealer portal → Account → Lead Notifications → Add Email
  • CarCostCanada → Member Tools → Lead Routing → Add CC
  • TAQ Auto → Settings → Lead Recipients

After 2–3 weeks of clean parallel running, Mil can simplify by REMOVING his old address from each provider — but most dealers keep both running indefinitely. Both are free; running both is the lowest-risk default.

Alternative if Mil prefers a single inbox: route from his existing dealership address (e.g. leads@huntchryslerfiat.ca) via Gmail / Workspace forwarding rule to huntchryslerfiat@firstfinancialcanada.com. Adds one hop, slightly slower delivery, but keeps his existing destination as the single source of truth.

Don't snoop for the existing address from the public website. Ask Mil during the kickoff call which approach he wants — most pick the dual-recipient option without hesitation.

Things to watch for

  • Cloudflare on huntchrysler.ca — same Convertus protection as Milton. The Inventory Importer's deep photo scan should clear ~70/79 vehicles per pass; the retry pass usually recovers most of the rest.
  • Stellantis Eshop conflict — Hunt's site uses the Stellantis e-shop widget. The importer correctly skips this in favor of the Convertus card data. No action needed.
  • VIN cross-listings — some "new" demo units appear in the Used section too. The ADD-mode sync will dedupe by VIN, so total count may be ~17% lower than (new + used) summed.
  • Lead intake email forwarding — Workspace blocks external forwarding by default. Verify Mil's IT has approved the forwarding rule from huntchrysler@firstfinancialcanada.com.
