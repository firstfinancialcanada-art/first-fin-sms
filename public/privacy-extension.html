First-Fin Browser Helper — Privacy Policy

Last updated: 2026-04-28 · Effective immediately

This policy describes how the First-Fin Browser Helper (the "Extension") handles data. The Extension is published by First Financial Canada ("First-Fin", "we", "us") to support its dealer-platform customers.

What the Extension does

The Extension is a single-purpose tool: it auto-fills the Facebook Marketplace "create vehicle listing" form with vehicle inventory data that is already loaded inside the user's authenticated First-Fin dealer-platform tab. The user clicks a button on the platform; the platform passes vehicle fields (year, make, model, price, mileage, description, photo URLs) to the Extension via in-page postMessage; the Extension opens or focuses a Facebook Marketplace tab and types those fields into the form. That is the entire flow.

Data the Extension reads

  • Vehicle data passed to it from app.firstfinancialcanada.com (the user's logged-in dealer dashboard). The user explicitly initiates this transfer by clicking the platform's "Auto-Fill Facebook" button.
  • Public images at the photo URLs the platform passes to it (we fetch them as base64 so Facebook's form will accept them).
  • The current state of the Facebook Marketplace listing form — only to determine which fields exist and where to type the values. We do not read existing form values, account data, messages, or anything else on Facebook.

Data the Extension does NOT read or transmit

  • Facebook account information, messages, friends list, ad activity, or browsing history
  • Other tabs, windows, or websites the user has open
  • Cookies, saved passwords, or autofill data from any site
  • Any personally identifiable information of customers, leads, or end-buyers
  • Analytics, telemetry, or usage tracking of any kind

Where data goes

Vehicle data flows in one direction only:

  1. From the First-Fin dealer platform tab (where the user is authenticated)
  2. Into the Extension's local memory
  3. Into the Facebook Marketplace form fields on the user's screen

The Extension does not transmit data to any third-party server, analytics provider, advertising network, or anywhere outside the user's own browser and the Facebook tab they are filling. There is no "phone home" call. Photo URLs are fetched directly from First-Fin's CDN (the same CDN the user's platform tab is already loading from).

Storage

The Extension uses Chrome's chrome.storage API only to cache short-lived state needed to coordinate the platform tab with the Facebook tab during a single auto-fill action. No persistent user data is stored. Uninstalling the Extension removes all its storage.

Authentication

The Extension itself does not authenticate. It relies on the user being signed in to app.firstfinancialcanada.com in their own browser tab. The Extension never sees the user's First-Fin password, JWT secret, or Facebook password.

Permissions and why we ask for them

  • tabs + scripting: needed to open or focus the Facebook Marketplace listing tab and inject the form-filler script into it.
  • storage: short-lived state caching during one auto-fill action (see "Storage" above).
  • Host access to app.firstfinancialcanada.com: to receive the auto-fill request the dealer platform sends from its own page.
  • Host access to facebook.com/marketplace/*: to fill the Marketplace listing form on the user's behalf.

Sharing & sale of data

We do not sell, rent, share, or trade any data the Extension touches with anyone. There is no advertising network integration, no analytics SDK, and no resale of vehicle or dealer information.

Children

The Extension is a B2B tool for licensed automotive dealers and their employees. It is not directed to anyone under 18 and we do not knowingly collect data from minors.

Changes to this policy

If we materially change how the Extension handles data, we will publish an updated version of this page at the same URL and update the "Last updated" date at the top.
